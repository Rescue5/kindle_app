# UI Change Rules

These rules are mandatory for all future interface work in this project.

- New features must fit into the existing information architecture.
- Do not create a separate visual style for each new feature.
- Reuse existing components before adding new components.
- Colors, sizes, spacing, radii, and animations must use design tokens.
- Do not add decorative panels or cards without functional need.
- Every interactive component must include hover, pressed, focused, disabled, and loading states.
- Every new screen must account for loading, empty, and error states.
- Business logic must not be mixed into the presentation layer.
- New features must look as if they were planned in the original application design.
- After major UI changes, run the application, capture a screenshot, and perform a visual polish pass.
- Keep the product calm, dense, and work-oriented; avoid generic AI-dashboard decoration.
